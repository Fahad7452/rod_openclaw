# Contributing

Thanks for contributing to EffectOS.

## Contribution Focus

Prioritize improvements that strengthen:
- Deterministic effect governance
- Policy clarity and enforcement guarantees
- Auditability and operational reliability

## Workflow

1. Fork or branch from `main`
2. Make focused changes
3. Add or update relevant documentation
4. Validate behavior with tests where available
5. Open a pull request with clear intent and scope

## Pull Request Checklist

- Purpose of change is clearly described
- Behavior and risk impact are explained
- Docs are updated if architecture, concepts, or usage changed
- No unrelated refactors included

## Documentation Standards

When updating docs:
- Keep terms consistent with `GLOSSARY.md`
- Explain both intent and enforcement behavior
- Prefer concrete examples over abstract claims
