# Versioning and Releases

## Versioning Approach

Use semantic-style versioning guidance:
- Major: breaking changes
- Minor: backward-compatible features
- Patch: fixes and non-breaking improvements

## Release Flow

1. Validate `docs/release-checklist.md`
2. Confirm high-risk tests pass
3. Update `CHANGELOG.md`
4. Tag release and publish notes

## Breaking Change Policy

Breaking behavior must include:
- Migration guidance
- Explicit mention in changelog
- Clear rollout and rollback strategy

## Release Notes Template

- Highlights
- Security or governance impact
- Breaking changes
- Migration notes
- Known limitations
