# ADR Template (Architecture Decision Record)

Use this template to track major technical decisions.

## Title

Short decision name.

## Status

Proposed | Accepted | Superseded

## Context

What problem are we solving?
What constraints matter (security, reliability, compliance, scale)?

## Decision

What was decided and why?

## Consequences

### Positive
- Expected benefits

### Negative
- Trade-offs and risks

## Alternatives Considered

1. Option A
2. Option B
3. Option C

## Effect on MEG Model

- Does this change runtime authority boundaries?
- Does this affect policy determinism?
- Does this affect audit trail completeness?
