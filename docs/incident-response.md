# Incident Response

## Purpose

Provide a structured response process for governance or execution incidents.

## Incident Types

- Suspected unauthorized execution
- Policy misconfiguration with production impact
- Audit trail integrity or availability issues

## Response Workflow

1. Detect and classify severity
2. Contain (limit high-risk effects if needed)
3. Investigate timeline and impacted scope
4. Remediate root cause
5. Recover and validate controls
6. Document lessons learned

## Evidence Checklist

- Relevant effect request IDs
- Authorization decision logs
- Policy version at incident time
- Affected systems and time window

## Post-Incident Actions

- Update runbook and tests
- Add guardrails for recurrence prevention
- Track action items to completion
