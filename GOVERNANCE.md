# Governance

## Governance Model

EffectOS follows a maintainer-led model with documented decision records.

## Decision Principles

- Preserve MEG boundaries (runtime proposes, MEG executes)
- Prefer deterministic policy and auditable outcomes
- Optimize for safety over convenience on high-risk effects

## Roles

- Maintainers: review, merge, and release changes
- Contributors: propose improvements via issues and pull requests

## Decision Process

1. Problem and options documented
2. Trade-offs discussed in issue/PR
3. Decision recorded using an ADR when architectural
4. Follow-up tasks tracked and completed
