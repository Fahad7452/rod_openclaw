# Security Policy

## Supported Scope

Security concerns are accepted for all active project materials in this repository, excluding archival content under `Old Files`.

## Reporting a Vulnerability

Please include:
- Vulnerability type
- Affected component or document path
- Reproduction steps
- Expected vs actual behavior
- Potential impact

## Response Expectations

Reports are triaged based on:
- Exploitability
- Impact on effect authorization or execution boundaries
- Audit integrity implications

## Important Context

EffectOS is a governance-oriented compute model.
Security controls are strongest when runtime execution authority remains externalized to MEG and policy is deny-by-default.
