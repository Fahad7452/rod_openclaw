# Glossary

## Computational Effect
An operation that changes system state in the machine world.

## Effect Materialization
The moment a proposed effect becomes an executed real-world state change.

## Runtime
The system component that reasons and proposes actions but does not own execution authority.

## Machine-Enforced Governance (MEG)
An externalized control plane that validates, authorizes, and executes effects deterministically.

## Authorization
A positive policy decision that permits an effect to execute.

## Deny by Default
Policy posture where effects are blocked unless explicitly allowed.

## Canonical Audit Trail (CAT)
A durable and consistent record of effect proposals, decisions, and execution outcomes.
