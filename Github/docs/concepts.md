# Concepts

## Computational Effects

A computational effect is any operation that changes system state.

Examples:
- Writing a file  
- Executing a command  
- Calling an API  
- Transferring value  

Reality changes only when an effect materializes.

## Machine-Enforced Governance (MEG)

MEG governs computational effects using deterministic, externalized validation.

- Positive authorization only  
- No execution authority in runtime  
- Canonical Audit Trail (CAT) required
