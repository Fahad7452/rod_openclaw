# Maintainers

## Maintainer Responsibilities

- Review pull requests for correctness and risk
- Protect MEG architecture boundaries
- Ensure policy and security implications are documented
- Keep release notes and changelog current
- Respond to critical issues promptly

## Review Standards

- Changes must have clear intent
- High-risk behavior must include tests or validation evidence
- Documentation must be updated when behavior changes

## Release Ownership

Maintainers coordinate release readiness using:
- `docs/release-checklist.md`
- `CHANGELOG.md`
